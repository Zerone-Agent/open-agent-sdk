# @codeany/open-agent-sdk 架构手册

## 项目概述

`@codeany/open-agent-sdk` 是一个**进程内运行的 AI Agent SDK**，无需依赖本地 CLI 子进程，可在云、Serverless、Docker、CI/CD 等任意环境部署。

- **包名**：`@codeany/open-agent-sdk`
- **版本**：0.2.0
- **最低 Node.js**：18.0.0
- **核心依赖**：`@anthropic-ai/sdk`、`@modelcontextprotocol/sdk`、`zod`

---

## 整体架构

项目采用 **4 层分层架构**，自上而下为：

```
Agent 层（入口 / 面向用户）
    ↓
Engine 层（核心 Agentic Loop）
    ↓
Provider 层（LLM 抽象）
Tool 层（30+ 内置工具）
```

辅以 **5 大辅助系统**：

- Skill 系统 — 可复用的提示词模板
- Hook 系统 — 生命周期钩子拦截
- Session 持久化 — 对话记录存盘
- MCP 集成 — 外部工具服务器
- 上下文压缩 — 自动摘要与截断

## 1. Agent 层 (`src/agent.ts`)

面向用户的高层 API，封装了所有配置和生命周期。

### 核心类：`Agent`

```typescript
const agent = createAgent({
  model: 'claude-sonnet-4-6',
  cwd: '/path/to/project',
  tools: ['BashTool', 'FileReadTool'],  // 可选：按名称筛选
})
```

#### 构造流程

```
Agent 构造
  ├─ 读取凭证（options / env / CODEANY_* 环境变量）
  ├─ 解析 API 类型（显式指定 > 环境变量 > 模型名启发式推断）
  ├─ 创建 LLM Provider（Anthropic / OpenAI）
  ├─ 初始化内置 Skills
  ├─ 构建 Hook 注册表
  ├─ 组装工具池（全部 / 按名称筛选 / 自定义 ToolDefinition[]）
  └─ 异步初始化（setup）
       ├─ 注册自定义 Agent 定义
       ├─ 连接 MCP 服务器（stdio / SSE / HTTP / 进程内 SDK Server）
       ├─ 恢复 Session（如配置了 resume）
       └─ 加载文件系统 Skills（如配置了 settingSources）
```

#### API 类型推断逻辑

当未显式指定 `apiType` 时，从模型名自动推断：

| 模型名特征 | 推断 API 类型 |
|-----------|-------------|
| 包含 `gpt-`、`o1`、`o3`、`o4`、`deepseek`、`qwen`、`glm`、`mistral`、`gemma` 等 | `openai-completions` |
| 其他（默认） | `anthropic-messages` |

#### 两种使用方式

**流式迭代** — 逐事件产出：

```typescript
for await (const event of agent.query('Hello')) {
  switch (event.type) {
    case 'assistant':     // LLM 响应
    case 'tool_result':   // 工具执行结果
    case 'result':        // 最终汇总
    case 'partial_message': // 流式文本片段
  }
}
```

**一次性返回** — 收集所有事件后返回最终结果：

```typescript
const result = await agent.prompt('Hello')
// result.text, result.usage, result.num_turns, result.duration_ms, result.messages
```

#### 生命周期

- 多轮对话通过 `this.history: NormalizedMessageParam[]` 持久化
- 每次 `query` 结束后更新 `this.history = engine.getMessages()`
- `close()` 方法持久化 Session 到磁盘并关闭 MCP 连接

## 2. Engine 层 (`src/engine.ts`)

整个 SDK 的**核心引擎**，实现了完整的 Agentic Loop。

### 核心循环

```
用户消息
  │
  ├─ 1. Hook: UserPromptSubmit（可阻断）
  ├─ 2. 写入 messages 数组
  ├─ 3. 构建工具列表（ToolDefinition → NormalizedTool）
  ├─ 4. 构建系统提示词（base + env + skills + AGENTS.md）
  └─ 5. 发出 SDKSystemMessage（init 事件）
       │
       ▼
  ┌─ Agentic Loop ─────────────────────────────┐
  │                                              │
  │  while (turnsRemaining > 0):                 │
  │    ├─ 检查 abort / budget                    │
  │    ├─ Auto-compact（如上下文超阈值）           │
  │    ├─ Micro-compact（截断大工具输出）           │
  │    ├─ 调用 LLM（流式 / 非流式 + retry）       │
  │    ├─ 累计 token / cost                      │
  │    ├─ 写入 assistant message                  │
  │    ├─ yield SDKAssistantMessage               │
  │    ├─ 处理 max_tokens 恢复（最多 3 次）        │
  │    ├─ 提取 tool_use blocks                    │
  │    ├─ 如无 tool_use → break（循环结束）        │
  │    ├─ 执行工具（只读并发 / 写操作串行）         │
  │    ├─ yield SDKToolResultMessage              │
  │    └─ 写入 tool results → 继续循环             │
  │                                              │
  └──────────────────────────────────────────────┘
       │
       ├─ Hook: Stop
       ├─ Hook: SessionEnd
       └─ yield SDKResultMessage（最终汇总）
```

### 系统提示词构建

```
basePrompt（默认预设 或 自定义）
  + envPrompt
      ├─ 可用 Subagents 列表
      ├─ <env> XML（模型、平台、日期、git 状态等）
      ├─ Skills XML 描述（供模型决策何时加载）
      └─ AGENTS.md 内容（项目级指令）
  + appendSystemPrompt（用户追加）
```

### 工具执行策略

```
toolUseBlocks
  │
  ├─ 按 isReadOnly() 分组
  │
  ├─ 只读工具 → Promise.all 并发（最多 10 个一批）
  │     └─ Glob, Grep, Read, WebFetch 等
  │
  └─ 写操作工具 → for await 串行
        └─ Write, Edit, Bash, Agent 等
```

每个工具执行前经过：

```
查找工具定义 → isEnabled 检查 → canUseTool 权限检查
  → Hook: PreToolUse（可阻断）
  → tool.call(input, context)
  → Hook: PostToolUse / PostToolUseFailure
```

### 流式 vs 非流式

| 模式 | 条件 | 特点 |
|------|------|------|
| 非流式 | `includePartialMessages = false`（默认） | 带 `withRetry` 指数退避重试 |
| 流式 | `includePartialMessages = true` | 逐 chunk yield `partial_message`，不支持重试 |

### 预算与安全控制

- **maxTurns**：最大循环轮次，默认 10
- **maxBudgetUsd**：最大花费金额，每次循环前检查
- **abortSignal**：外部中断

## 3. Provider 层 (`src/providers/`)

### 统一抽象接口

```typescript
interface LLMProvider {
  readonly apiType: ApiType
  createMessage(params: CreateMessageParams): Promise<CreateMessageResponse>
  createMessageStream?(params: CreateMessageParams): AsyncGenerator<StreamChunk>
}
```

### 内部数据格式

SDK 内部使用 **Anthropic-like 消息格式**作为规范化表示：

```typescript
interface NormalizedMessageParam {
  role: 'user' | 'assistant'
  content: string | NormalizedContentBlock[]
}

type NormalizedContentBlock =
  | { type: 'text'; text: string }
  | { type: 'tool_use'; id: string; name: string; input: any }
  | { type: 'tool_result'; tool_use_id: string; content: string; is_error?: boolean }
  | { type: 'image'; source: any }
  | { type: 'thinking'; thinking: string }
```

### 两个实现

| Provider | 文件 | 特点 |
|----------|------|------|
| `AnthropicProvider` | `src/providers/anthropic.ts` | 薄封装，直接透传（内部格式即 Anthropic 格式） |
| `OpenAIProvider` | `src/providers/openai.ts` | 格式转换层，OpenAI Chat Completions ↔ Normalized |

工厂函数 `createProvider(apiType, opts)` 按需创建。

## 6. Hook 系统 (`src/hooks.ts`)

生命周期钩子，支持拦截和扩展 Agent 行为。

### 支持的事件（20 个）

| 事件 | 触发时机 |
|------|---------|
| `SessionStart` | 会话开始 |
| `SessionEnd` | 会话结束 |
| `UserPromptSubmit` | 用户发送消息（可阻断） |
| `PreToolUse` | 工具执行前（可阻断） |
| `PostToolUse` | 工具执行成功后 |
| `PostToolUseFailure` | 工具执行失败后 |
| `Stop` | 循环结束 |
| `SubagentStart` / `SubagentStop` | 子 Agent 生命周期 |
| `PreCompact` / `PostCompact` | 上下文压缩前后 |
| `PermissionRequest` / `PermissionDenied` | 权限相关 |
| `TaskCreated` / `TaskCompleted` | 任务生命周期 |
| `ConfigChange` / `CwdChanged` / `FileChanged` | 状态变更 |
| `Notification` | 系统通知 |
| `TeammateIdle` | 团队成员空闲 |

### Hook 定义

```typescript
interface HookDefinition {
  command?: string          // Shell 命令（通过 bash -c 执行）
  handler?: (input) => ...  // 函数处理器
  matcher?: string          // 工具名正则匹配
  timeout?: number          // 超时（默认 30s）
}
```

### Hook 输出

```typescript
interface HookOutput {
  block?: boolean           // 阻断操作
  message?: string          // 追加到对话
  permissionUpdate?: { tool: string; behavior: 'allow' | 'deny' }
  notification?: { title: string; body: string; level?: 'info' | 'warning' | 'error' }
}
```

### 执行模型

1. 函数处理器：`Promise.race([handler(input), timeout])`
2. Shell 命令：通过 `spawn('bash', ['-c', command])` 执行，stdin 传入 JSON，stdout 解析为 HookOutput

## 8. MCP 集成 (`src/mcp/client.ts`)

### 连接类型

| 类型 | 配置 | 说明 |
|------|------|------|

### 集成流程

```

  └─ 遍历 mcpServers 配置
       ├─ SDK Server → 直接将 tools 合入 toolPool

            ├─ 发现工具
            ├─ 转换为 ToolDefinition
            └─ 合入 toolPool
```

## 9. 上下文压缩 (`src/utils/compact.ts`)

三级压缩机制，应对长对话上下文溢出。

### Auto-Compact（自动压缩）

```
每次循环前检查：
  estimateMessagesTokens(messages) >= getAutoCompactThreshold(model)
    ↓ 是
  调用 LLM 对话摘要 → 替换 messages 为压缩版
```

- 状态追踪：`AutoCompactState { compacted, turnCounter, consecutiveFailures }`
- 连续失败 3 次后停止尝试
- prompt 超长时也会触发紧急压缩

### Micro-Compact（微压缩）

每轮 API 调用前，截断大型工具结果输出，减少单次请求体积。

### 上下文窗口阈值

根据模型自动选择（如 Claude 系列约 200k tokens），预留缓冲区。

## 10. 重试机制 (`src/utils/retry.ts`)

```typescript
withRetry(fn, config?, abortSignal?)
```

- **可重试错误**：速率限制（429）、服务过载（529）、超时
- **不可重试错误**：认证错误、prompt 过长
- **策略**：指数退避 + 抖动
- **默认配置**：最大重试次数、初始延迟、最大延迟

## 数据流总览

```
用户输入
  │
  ▼
Agent.query(prompt)
  │
  ▼
QueryEngine.submitMessage(prompt)
  │
  ├─→ buildSystemPrompt() ──→ [base + env + skills + AGENTS.md]
  │
  ├─→ provider.createMessage(system, messages, tools)
  │       │
  │       ├─ AnthropicProvider ──→ Anthropic API
  │       └─ OpenAIProvider ──→ OpenAI API
  │
  ├─← CreateMessageResponse (text + tool_use blocks)
  │
  ├─→ executeTools(toolUseBlocks)
  │       ├─ 只读 → Promise.all 并发
  │       └─ 写操作 → 串行执行
  │
  ├─← ToolResult[]
  │
  └─→ yield SDKMessage events → 用户
```
